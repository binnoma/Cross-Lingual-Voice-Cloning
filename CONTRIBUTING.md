## 📋 قواعد المساهمة

### 1. الإبلاغ عن الأخطاء (Bug Reports)

قبل الإبلاغ عن خطأ، يرجى:
- التأكد من أن الخطأ لم يُبلغ عنه مسبقاً في [Issues](../../issues)
- استخدام قالب الإبلاغ عن الأخطاء
- تضمين خطوات إعادة الإنتاج بالتفصيل

**قالب الإبلاغ:**
```markdown
## وصف الخطأ
وصف واضح ومختصر للخطأ

## خطوات الإعادة
1. اذهب إلى '...'
2. اضغط على '...'
3. مرر لأسفل إلى '...'
4. شاهد الخطأ

## السلوك المتوقع
وصف ما كنت تتوقع أن يحدث

## لقطات الشاشة
إن أمكن، أضف لقطات شاشة

## بيئة التشغيل
- نظام التشغيل: [مثال: Windows 11]
- المتصفح: [مثال: Chrome 120]
- إصدار Node.js: [مثال: 20.10]
- إصدار Python: [مثال: 3.11]
```

### 2. طلب الميزات (Feature Requests)

نرحب باقتراحاتكم! يرجى:
- وصف الميزة المطلوبة بالتفصيل
- شرح حالات الاستخدام
- تقديم أمثلة إن أمكن

### 3. إرسال التعديلات (Pull Requests)

#### العملية:

1. **Fork** المشروع
2. أنشئ فرع جديد (`git checkout -b feature/amazing-feature`)
3. نفذ التعديلات (`git commit -m 'إضافة ميزة رائعة'`)
4. ادفع الفرع (`git push origin feature/amazing-feature`)
5. افتح **Pull Request**

#### قواعد الكود:

| القاعدة | التفصيل |
|---------|---------|
| **TypeScript** | استخدم TypeScript صارم في جميع ملفات الواجهة |
| **Python** | اتبع PEP 8 مع type hints كاملة |
| **التسمية** | استخدم أسماء وصفية بالإنجليزية |
| **التعليقات** | أضف تعليقات توضيحية بالعربي والإنجليزي |
| **الاختبار** | اختبر الكود قبل الإرسال |
| **الالتزام** | رسائل git واضحة بالعربي أو الإنجليزي |

#### هيكل رسالة الالتزام:
```
نوع: وصف مختصر

وصف تفصيلي إن لزم الأمر
```

**الأنواع:**
- `إضافة:` ميزة جديدة
- `إصلاح:` تصحيح خطأ
- `تحسين:` تحسين أداء أو كود
- `توثيق:` تغييرات في التوثيق
- `نمط:` تنسيق الكود

## 🏗️ إعداد بيئة التطوير

### المتطلبات:
- Node.js 18+
- Python 3.10+
- Git
- Bun (مُوصى به)

### الخطوات:
```bash
# 1. Fork و Clone
git clone https://github.com/YOUR_USERNAME/Cross-Lingual-Voice-Cloning.git
cd Cross-Lingual-Voice-Cloning

# 2. تثبيت تبعيات الواجهة
npm install

# 3. تثبيت تبعيات خدمة الصوت
cd mini-services/voice-service
pip install -r requirements.txt

# 4. إعداد البيئة
cp .env.example .env.local

# 5. تشغيل التطبيق
npm run dev
```
